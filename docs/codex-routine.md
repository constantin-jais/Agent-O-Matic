# Codex Routine

`cosmatic` is the repeatable harness for Codex sessions. Use it to move durable
instructions out of memory and into checked project files.

## Install locally

From the repository root:

```sh
cargo install --path crates/cli
cosmatic --help
```

After changing the CLI crate, reinstall before relying on the global command.
During development, the equivalent no-install fallback is:

```sh
cargo run -q --bin cosmatic -- <command>
```

## Start a session

Run these before making non-trivial changes:

```sh
git status --short --branch
cosmatic generate --check --manifest harness.toml
cosmatic goals --manifest harness.toml
cosmatic inspect adr-required --root .
cosmatic inspect language-ownership --root .
cosmatic inspect frontend-strict --root .
cosmatic inspect shell-debt --root .
```

If a repository does not have `harness.toml`, initialize one with:

```sh
cosmatic init --name <repo-name> --level L0 --adapter universal --yes
```

Then review the generated manifest and commit the generated agent configuration.

## Before implementation

- Read the relevant ADRs before architecture changes.
- Add or update tests before production code for non-trivial logic.
- Add a new ADR when `cosmatic inspect adr-required` flags a durable exception.
- Keep secrets out of manifests, generated files, logs, and docs.

## Before completion

Run the full local gate:

```sh
cargo fmt --all --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
cosmatic generate --check --manifest harness.toml
cosmatic goals --manifest harness.toml
```

For repositories with `.cosmatic` policies generated by `cosmatic init`, also run:

```sh
cosmatic inspect adr-required --root . --policy .cosmatic/adr-required.toml
cosmatic inspect language-ownership --root . --policy .cosmatic/language-ownership.toml
cosmatic inspect frontend-strict --root . --policy .cosmatic/frontend-strict.toml
cosmatic inspect shell-debt --root . --policy .cosmatic/shell-debt.toml
```

## Autonomous loop

Use `cosmatic loop --dry-run` as the routine check. Live mode is not a routine
developer command.

Live loop constraints:

- sandbox repository only;
- `cosmatic_SANDBOX=true` set on that repository;
- scoped service credential, never a broad human token;
- deploy commands are explicit environment variables;
- `cosmatic_LOOP_DISABLED` remains the emergency stop.

The helper script preserves that split:

```sh
export GITHUB_TOKEN="$(gh auth token)"
scripts/live-loop-smoke.sh <issue-number> "smoke title"
```

The real path requires the explicit `cosmatic_LIVE=1` opt-in and must only target
a disposable sandbox repository.
